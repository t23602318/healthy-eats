# Requirements Document

## Introduction

HealthyEats 是一款移动端 H5 健康饮食管理应用。用户通过拍照上传食物图片，AI（通义千问视觉模型）自动识别食物并估算营养成分，用户确认后记录到每日饮食日志中。App 提供个人档案管理、每日热量/营养素追踪、智能饮食建议等功能，帮助用户实现健康饮食与体重管理目标。

## Glossary

- **App**：HealthyEats H5 应用
- **User**：使用本 App 的终端用户
- **Account**：用户注册的账号（账号名 + 密码），数据与账号绑定
- **AI 识别**：通过通义千问视觉模型（qwen-vl-plus）分析食物图片并返回营养估算
- **BMR**：基础代谢率（Basal Metabolic Rate）
- **TDEE**：总每日能量消耗（Total Daily Energy Expenditure）
- **实际摄入热量**：AI 预估总热量 × 用户填写的实际摄入百分比
- **热量差值**：TDEE - 当日实际摄入热量总和

## Requirements

### Requirement 1: 账号注册与登录

**User Story:** 作为用户，我想注册一个账号并登录，让我的数据与账号绑定，在同一设备上切换账号可查看不同数据。

#### Acceptance Criteria

1. THE App SHALL 提供独立的登录页面（login.html），包含登录和注册两个 Tab。
2. 注册时，用户输入账号名（≥2 字符）和密码（≥4 字符），注册成功后自动登录。
3. 登录时，验证账号和密码，验证通过则登录成功跳转首页，验证不通过显示具体错误信息。
4. 已登录状态下，用户可在"我的"页面退出登录，退出后跳转到登录页。
5. 未登录状态访问任何功能页面，自动跳转到登录页。
6. 所有数据存储以账号名为 key 前缀，不同账号数据严格隔离。

---

### Requirement 2: 用户档案管理

**User Story:** 作为用户，我想维护身高、体重等身体指标，系统据此计算每日营养目标。

#### Acceptance Criteria

1. 档案表单包含：性别、年龄、身高、体重、体脂率（选填）、活动水平（5 档）、健康目标（减脂/维持/增肌）。
2. 表单验证：年龄 1–120、身高 50–250 cm、体重 10–500 kg、体脂率 0–70%。
3. 档案页面默认只读模式，以文本列表形式展示各字段值。点击"编辑"进入编辑模式，编辑模式显示"保存"和"取消"按钮。
4. 保存成功后自动计算并展示 BMI、TDEE、每日蛋白质目标等统计数据。
5. 支持上传个人头像（圆形裁剪），头像存储在账号数据中。

---

### Requirement 3: AI 拍照识别食物

**User Story:** 作为用户，我想拍照或从相册选择食物图片，AI 自动识别食物种类并估算营养成分。

#### Acceptance Criteria

1. 记录饮食页面提供"拍照/选图"入口，支持手机相机拍照和本地相册选择。
2. 用户选择图片后，自动调用通义千问视觉 API 进行食物识别，期间显示加载状态。
3. 识别成功后展示：食物列表（名称、重量、热量）、营养汇总（总热量、蛋白质、碳水、脂肪）。
4. 用户可通过滑块或输入框调整实际摄入百分比（1–100%），实时计算实际摄入热量和营养素。
5. 用户选择餐次（早餐/午餐/晚餐）后点击"记录到餐次"保存。
6. 如果 AI 识别失败或未检测到食物，显示友好错误提示并支持重试。

---

### Requirement 4: 首页今日热量与营养总览

**User Story:** 作为用户，我想在首页直观看到今日热量和营养素摄入情况。

#### Acceptance Criteria

1. 首页展示"今日热量总览"卡片：最多可摄入（TDEE）、已摄入、还可摄入（可为负数，超标时高亮）。
2. 展示碳水、蛋白质、脂肪三种营养素的目标值、已摄入值、剩余值。
3. 展示三餐卡片（早/午/晚），每餐显示实际/目标热量和进度条。
4. 首页底部展示"今日建议"区块，基于当前摄入与目标的对比给出营养建议。
5. 如果未完善档案，显示提示引导用户去填写。

---

### Requirement 5: 饮食记录管理

**User Story:** 作为用户，我想查看和管理所有饮食记录。

#### Acceptance Criteria

1. 支持按日和按周两种维度查看记录。
2. 日视图：日期导航、当日统计（摄入/目标/差值）、记录列表（缩略图、食物名、餐次、热量、时间）。
3. 周视图：最近 7 天每天汇总行，可展开查看详情。
4. 支持删除记录（确认弹窗后删除，实时刷新数据）。
5. 空状态提示引导用户去记录饮食。

---

### Requirement 6: 数据持久化与账号隔离

**User Story:** 作为用户，我希望数据在关闭浏览器后仍保留，且不同账号的数据互不干扰。

#### Acceptance Criteria

1. 所有 localStorage 键以 `{username}_` 为前缀，实现账号级数据隔离。
2. 档案键：`{username}_health_profile`，饮食记录键：`{username}_diet_log_YYYY-MM-DD`。
3. 读取数据时对损坏 JSON 执行降级处理（删除损坏键，返回默认值）。

---

### Requirement 7: 移动端 H5 适配

**User Story:** 作为用户，我想在手机浏览器上流畅使用本 App。

#### Acceptance Criteria

1. 界面标题显示"HealthyEats"。
2. 响应式布局支持 320px–430px 屏幕，不出现横向滚动条。
3. 所有可交互元素触摸区域 ≥ 44×44 CSS 像素。
4. 底部导航包含 3 个 Tab：首页、记录、我的（扁平 SVG 图标）。
5. 右下角悬浮操作按钮（FAB）快速进入记录饮食页面。
6. 星巴克风格 UI 设计：深绿主色、金色强调、奶白背景、药丸形按钮、大圆角卡片。
